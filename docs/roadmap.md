# Roadmap

This roadmap is a directional planning document for maintainers and future
stewards.

It is not a guarantee. It is a transparent statement of where the project is
trying to go next.

## Roadmap Principles

- prioritize beneficiary safety over feature novelty
- prioritize operational usefulness over model complexity for its own sake
- prioritize maintainability and handoff readiness
- prioritize evidence generation over inflated claims

## Near-Term Priorities

### 1. Publication and steward readiness

- complete public metadata cleanup
- replace placeholder URLs and contact details
- prepare a public issue-triage and support model
- finalize the public-good package for publication

### 2. Real partner-data validation

- secure partner exports
- run partner readiness suites
- assess fairness and precision-at-capacity on real program data
- generate shadow-mode evidence

### 3. Workflow adoption depth

- continue strengthening embedded operations into upstream systems
- improve field-facing simplicity
- continue reducing friction in the action and verification loop

### 4. Reliability and maintenance

- continue frontend modularization
- address dependency warnings and vulnerabilities
- deepen test coverage and release discipline

## Medium-Term Priorities

- stronger deployment verification on real infrastructure
- clearer steward support model
- more real-world evaluation evidence
- broader connector maturity where there is real user demand

## Out Of Scope Or Deliberately Deferred

- automated beneficiary exclusion decisions
- marketing the model as universally valid across contexts
- claiming impact based only on public benchmarks
- replacing field staff judgment with automation
