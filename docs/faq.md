# FAQ

## Is RetainAI a case-management system?

No. It is a retention operations layer designed to work alongside existing systems.

## Does RetainAI replace field staff judgment?

No. The product is designed for human-in-the-loop supportive follow-up.

## Can it be used to remove beneficiaries?

It should not be used that way. The repository documentation explicitly treats punitive or exclusionary use as out of scope and harmful.

## Does it require cloud hosting?

No. The project is designed for self-hosting and includes local and self-managed deployment paths.

## Is the model universal across all contexts?

No. The platform is built around local validation, local fine-tuning, and explicit evidence review.

## Can we run it without rich household survey data?

Yes. The platform is designed to work with narrow operational data, especially repeated interaction histories.

## Does the repository include evaluation tooling?

Yes. It includes backtesting, partner-readiness checks, cross-segment validation, synthetic stress testing, and shadow-mode capture.

## Is it already proven in live NGO deployments?

Not by the repository alone. Real-world deployment evidence depends on partner data and field use.
