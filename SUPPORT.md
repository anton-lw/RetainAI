# Support and Maintenance

## Current State

RetainAI is an open-source project intended for public-good stewardship and eventual handoff. Support and maintenance may therefore vary depending on the current steward and the deployment context.

## Repository-Level Support

At the repository level, the expected support channels are:

- documentation
- issue tracking at [github.com/anton-lw/retainai/issues](https://github.com/anton-lw/retainai/issues)
- security reporting through the route documented in [SECURITY.md](SECURITY.md)
- contribution review through the workflow described in [CONTRIBUTING.md](CONTRIBUTING.md)

The current repository steward and support contact is Anton Waagaard
(`hello@antons.site`).

For general repository support:

- use GitHub issues for bugs, feature requests, and documentation problems
- use `hello@antons.site` for steward contact or support requests that should
  not begin as a public issue

## Deployment-Level Support

Each organization deploying RetainAI is responsible for:

- infrastructure operations
- user administration
- connector configuration
- local data governance
- model validation and approval
- incident response

## What the Software Includes

The repository includes:

- operational runbooks
- validation scripts
- smoke-check tooling
- seeded development data
- browser and backend test coverage

These assets are designed to reduce support burden and make stewardship transfer easier.

## What the Software Does Not Promise

The open-source project does not guarantee:

- a hosted support desk
- automatic compliance in any jurisdiction
- universal model validity across sectors or countries
- out-of-the-box production readiness without deployment review

## Recommended Handoff Package

Any institutional handoff should preserve:

- the license
- governance and safety documents
- operational runbooks
- validation workflow
- a public issue tracker
- a named maintainer or steward
